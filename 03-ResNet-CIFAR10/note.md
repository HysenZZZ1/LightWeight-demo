## 一、本次实验新的收获（相关知识点补充）
### 1. 为什么残差块里的第二个卷积层不能先 ReLU？
在经典的 ResNet 基础块（BasicBlock）中，标准结构为：
输入 x → 卷积1 → BN → ReLU → 卷积2 → BN → 残差相加 → ReLU → 输出

**第二个卷积后、相加前不使用 ReLU 的原因：**
1. **保证残差可以取负值**
   如果第二个卷积后直接加 ReLU，输出会被限制为非负数，导致残差分支只能学习正向增量，无法修正负向贡献，严重限制网络表达能力。
2. **更容易学习恒等映射**
   不加激活时，若卷积权重趋近于 0，残差块直接输出输入 x，完美实现恒等映射；若添加 ReLU，会截断负梯度，导致深层网络难以优化。
3. **保证梯度顺畅流动**
   相加后再统一激活，能让残差分支保持线性路径，避免非线性截断梯度，符合 ResNet 核心设计思想。

---

### 2. downsample 是干嘛的？
downsample 是**残差块内部的维度适配模块**，作用是让残差分支 x 与主分支输出 out 能够正常相加。

#### （1）为什么需要 downsample？
- 主分支经过卷积后，**通道数**或**特征图尺寸**会发生变化
- 残差分支仍保持原始形状，与主分支输出不匹配，无法直接执行 `out += residual`

#### （2）downsample 的核心功能
- 调整通道数：使用 **1×1 卷积** 实现通道升维/降维
- 调整尺寸：使用与主分支相同的 stride 同步下采样
- 标准实现：
  ```python
  downsample = nn.Sequential(
      nn.Conv2d(in_ch, out_ch, kernel_size=1, stride=stride, bias=False),
      nn.BatchNorm2d(out_ch)
  )
  ```

## 二、本次实验中出现的问题
### 1. 通道不匹配问题
网络层与层之间的通道数必须连贯，前一层的输出通道 = 后一层的输入通道，否则卷积层无法接收输入，直接报错。

### 2. 为什么 downsample 没有解决第一层卷积的通道匹配问题？
**downsample 只负责解决【残差块内部】的维度不匹配**，无法解决【网络层之间】的输入通道错误。
如果第一层卷积的输入通道就不匹配，模型还没执行到 downsample 就会直接崩溃。

### 3. downsample 使用注意事项
- 必须使用 **1×1 卷积**，不能用 3×3 卷积
- stride 必须与主分支下采样的 stride **完全一致**
- 必须配合 BN 层保证训练稳定

## 三、实验总结
在 ResNet-CIFAR10 实验中：
1. **通道数必须前后连贯**，这是模型能运行的基础
2. **特征图尺寸必须匹配**：3×3 卷积必须设置 `padding=1`
3. **downsample 是块内适配器**，仅用于解决残差相加前的维度不匹配问题
4. 1×1 卷积 + 同步 stride 是保证尺寸/通道匹配的关键
